# LlamaFactory - 大白话版本

> 📄 **原论文**: [LlamaFactory: Unified Efficient Fine-Tuning of 100+ Language Models](../pdfs/LlamaFactory_2403.13372.pdf)
>
> 🎓 **一句话概括**: 一个让普通人也能用游戏显卡微调大模型的"万能工具箱"

---

## 这个项目是干嘛的？

**微调**就是把通用大模型（比如 ChatGPT）变成专业助手（比如医疗 AI、法律 AI）。

以前这事儿很麻烦：
- 要懂很多技术
- 要租昂贵的云服务器
- 每换个模型就要重写一堆代码

**LlamaFactory 就像"AI 定制工厂"**——选好模型、准备好数据、点几下鼠标，就能得到你专属的 AI。

## 为什么需要这个？

### 问题：微调大模型太难了

| 难点 | 传统方式 | LlamaFactory 解决 |
|------|----------|-------------------|
| **技术门槛** | 要写很多代码，懂各种库 | Web 界面点几下就行 |
| **资源需求** | 要租多卡云服务器（贵） | 用你的游戏显卡就能跑 |
| **模型适配** | 每换个模型要重写代码 | 支持 100+ 模型，一键切换 |
| **方法选择** | 每种方法要单独学习 | 集成 10+ 方法，自动适配 |

### 解决思路

> 把微调流程拆成三块（模型、数据、训练），
>
> 每块做成"即插即用"的模块，
>
> 再加个网页界面，让普通人也能操作。

## 它是怎么工作的？

### 核心架构（3 分钟看懂）

```
┌─────────────────────────────────────────────────────────────┐
│                  你（用户）                                  │
│              LLAMABOARD 网页界面                             │
│                                                             │
│  "我要微调 Llama3-8B，用 LoRA，数据是我的医疗问答..."         │
│                                                             │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │ 选模型      │ │ 上传数据    │ │ 选训练方法  │           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
└─────────────────────────────┬───────────────────────────────┘
                              │ 一键启动
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    LlamaFactory 后台                         │
│                                                             │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │ Model Loader│ │ Data Worker │ │   Trainer   │           │
│  │             │ │             │ │             │           │
│  │ 模型加载    │ │ 数据处理    │ │ 训练执行    │           │
│  │ 量化压缩    │ │ 格式统一    │ │ 方法应用    │           │
│  │ 挂载适配器  │ │ 序列打包    │ │ 参数优化    │           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
│                                                             │
└─────────────────────────────┬───────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                 你的定制 AI 模型                              │
│                                                             │
│    "我是专业的医疗 AI，能回答您的健康问题..."                  │
└─────────────────────────────────────────────────────────────┘
```

### 三大模块

| 模块 | 干什么 | 通俗解释 |
|------|--------|----------|
| **Model Loader** | 处理模型 | 把模型"变小"（量化）、"改装"（挂适配器） |
| **Data Worker** | 处理数据 | 把各种格式的数据"标准化"，变成模型能吃的样子 |
| **Trainer** | 执行训练 | 用各种高效方法（LoRA、QLoRA 等）来训练 |

### 关键技术：让训练变"轻"

| 技术 | 作用 | 效果 |
|------|------|------|
| **LoRA** | 只训练一小部分参数 | 省内存、速度快 |
| **QLoRA** | LoRA + 4 位量化 | 内存降到原来的 4%！ |
| **Flash Attention** | 快速计算注意力 | 再提速 2-3 倍 |

---

## 效果怎么样？

### 内存需求对比

| 方法 | 内存占用（7B 模型） | 对比 |
|------|---------------------|------|
| 全量微调 | **38 GB** | 需要 A100 |
| Freeze 微调 | 15 GB | 还是很高 |
| LoRA | 16 GB | 可以用 3090 |
| **QLoRA** | **7.5 GB** | **用 3060 就能跑！** |

### 小模型更夸张

| 模型 | QLoRA 内存 | 你能用的显卡 |
|------|------------|--------------|
| Gemma-2B | **5.2 GB** | RTX 2060 就行 |
| Llama2-7B | **7.5 GB** | RTX 3060 12GB |
| Llama3-8B | ~8 GB | RTX 3070 |

**一句话**：用家用显卡就能微调大模型！

### 效果好不好？

在文本生成任务上，**LoRA 和 QLoRA 效果最好**，甚至比全量微调还好！

## 有什么缺点？

1. **有些模型不支持某些方法**：比如 Gemma 不支持 GaLore
2. **量化模型只能用适配器方法**：不能全量微调
3. **超大模型还是不行**：70B 模型还是要多卡
4. **多模态支持有限**：主要支持文本，图片支持较少

## 关键概念速查

| 术语 | 人话解释 |
|------|----------|
| **微调（Fine-tuning）** | 把通用 AI 变成专业 AI |
| **LoRA** | 只改模型一小部分，不动大参数 |
| **QLoRA** | LoRA + 压缩，更省内存 |
| **量化** | 把模型"压缩"，从 16 位变成 4 位 |
| **适配器** | 加在模型上的"小插件"，方便修改 |
| **RLHF** | 用人类反馈训练，让 AI 更听话 |

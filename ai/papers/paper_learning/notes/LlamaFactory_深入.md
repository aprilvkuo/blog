## What - 这是什么？

### 核心定义

LlamaFactory 是一个**统一高效微调框架**，通过模块化设计集成 10+ 种前沿微调方法，支持 100+ 不同架构的 LLM，提供命令行和 Web 界面实现零代码/少代码微调。

### 核心组件详解

| 组件 | 功能 | 设计理由 |
|------|------|----------|
| **Model Loader** | 模型初始化、量化、适配器挂载 | 抽象模型差异，统一接口 |
| **Data Worker** | 数据加载、对齐、预处理 | 统一数据格式，支持多来源 |
| **Trainer** | 训练方法、高效微调、计算优化 | 即插即用，快速集成新方法 |
| **Model 注册表** | 精确定位适配器挂载层 | 避免手动查找，支持新模型 |
| **数据描述规范** | 定义数据格式映射 | 不同数据集格式自动转换 |

## Why - 为什么重要？

### 研究缺口分析

| 缺口类型 | 现有问题 | LlamaFactory 解决 |
|----------|----------|-------------------|
| **方法覆盖** | 各框架只支持部分方法 | 集成 10+ 种方法 |
| **模型兼容** | 新模型需重新适配 | 模型注册表 + 自动挂载 |
| **用户门槛** | 需要编码实现 | Web UI 零代码 |
| **训练流程** | SFT/RLHF/DPO 分散 | 统一支持全流程 |

### 核心命题验证

| 命题 | 假设 | 验证结果 |
|------|------|----------|
| P1 | 模块化设计可实现即插即用 | ✅ 100+ 模型无缝切换 |
| P2 | QLoRA 可大幅降低内存 | ✅ 从 18 bytes → 0.6 bytes |
| P3 | Web UI 可降低使用门槛 | ✅ LLAMABOARD 可用 |
| P4 | 单模型 RLHF 可行 | ✅ Model-Sharing RLHF |

### 学术价值

1. **首次统一微调框架**：整合预训练、SFT、RLHF、DPO 全流程
2. **Model-Sharing RLHF 创新**：单模型动态切换多角色
3. **模块化设计范式**：为后续框架设计提供参考

### 技术实现细节

#### 1. 模型注册表

```python
# 注册模型适配器位置
MODEL_REGISTRY = {
    "llama": ["q_proj", "k_proj", "v_proj", "o_proj"],
    "chatglm": ["query_key_value", "dense"],
    "qwen": ["c_attn", "attn.c_proj"],
    # ... 自动扩展新模型
}

def find_all_linear_names(model):
    # 自动查找所有线性层
    return [name for name, module in model.named_modules()
            if isinstance(module, Linear)]
```

#### 3. Model-Sharing RLHF

```python
# 单模型动态切换角色
class ModelSharingRLHF:
    def __init__(self, model):
        # 一个模型，四个角色
        self.model = model  # 基础模型

        # 添加不同适配器
        self.model.add_adapter("policy", LoRAConfig())
        self.model.add_adapter("value", LoRAConfig())
        self.model.add_adapter("ref", LoRAConfig())  # frozen
        self.model.add_adapter("reward", LoRAConfig())

    def step(self, batch):
        # 动态切换适配器
        self.model.set_adapter("policy")
        policy_output = self.model(batch)

        self.model.set_adapter("value")
        value_output = self.model(batch)
```

#### 4. 高效微调方法集成

| 方法 | 核心思想 | 适用场景 |
|------|----------|----------|
| **LoRA** | 低秩分解，只训练 A×B | 通用微调 |
| **QLoRA** | 4 位量化 + LoRA | 低资源环境 |
| **DoRA** | 分解权重为幅度+方向 | 更精细控制 |
| **LoRA+** | LoRA 的非对称改进 | 更快收敛 |
| **PiSSA** | 主成分初始化 | 更高起点 |
| **GaLore** | 低秩梯度投影 | 大模型优化 |
| **BAdam** | 分块 Adam | 内存优化 |

### 性能结果

| 方法 | Gemma-2B | Llama2-7B | 内存效率 |
|------|----------|-----------|----------|
| Full-tuning | 17.06 GB | 38.72 GB | 18 bytes/param |
| Freeze | 8.10 GB | 15.69 GB | 8 bytes/param |
| LoRA | 7.91 GB | 16.32 GB | 1.8 bytes/param |
| **QLoRA** | **5.21 GB** | **7.52 GB** | **0.6 bytes/param** |

**关键洞察**：QLoRA 内存降低 96%！

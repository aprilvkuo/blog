# 常见问题

Paper Reading Skill 的常见问题解答。

---

## 什么时候用大白话版本？什么时候用深入版本？

| 场景 | 推荐 |
|------|------|
| 第一次接触论文 | 大白话版本 |
| 需要快速了解核心思想 | 大白话版本 |
| 技术细节分析 | 深入版本 |
| 写论文/做实验参考 | 深入版本 |
| 复习已读论文 | 根据需求选择 |

**建议**: 每篇论文都产出双版本，相互补充。

---

## 论文太长，读不完怎么办？

1. **优先级阅读**:
   - Abstract → Introduction → Methods → Results → Conclusion
   - 跳过 Appendix（除非需要细节）

2. **分层产出**:
   - 大白话版本：只看 Abstract + Introduction
   - 深入版本：补充 Methods + Results

3. **后续补全**:
   - 先产出框架笔记
   - 需要时再补充细节

## 如何判断理解程度？

| 理解程度 | 标志 |
|----------|------|
| 30% | 知道论文讲什么问题 |
| 50% | 能复述核心方案 |
| 70% | 能解释技术细节 |
| 80% | 能回答理解检验问题 |
| 90% | 能批判性思考 + 实践应用 |
| 100% | 能写相关论文/扩展实验 |

---

## 笔记写多少字合适？

| 版本 | 建议字数 |
|------|----------|
| 大白话 | 500-1000 字 |
| 深入 | 1500-3000 字 |
| 原子概念 | 200-500 字 |

**原则**: 精简胜于冗长，关键是结构完整。

---

## 如何跨论文关联？

1. **原子概念**: 发现重复出现的概念，存入 `insights/`
2. **对比笔记**: 多篇论文同一主题，创建 `{主题}-对比.md`
3. **wikilinks**: 笔记中用 `[[论文]]` 引用其他论文

示例：
```markdown
## 相关论文

- [[OpenDevin]]: 同样使用 Event Stream
- [[AgentScope]]: 类似的分布式架构
- [[SWE-Agent]]: 对比基线
```

## 理解检验问题怎么设计？

| 问题类型 | 示例 |
|----------|------|
| 定义理解 | "什么是 Event Stream？" |
| 设计理由 | "为什么用 Docker 而非直接执行？" |
| 对比分析 | "Event Stream 与消息队列的区别？" |
| 限制分析 | "这个方案的适用边界是什么？" |

## 批判性思考怎么写？

好的批判性思考应该：
- 质疑假设，而非否定结论
- 提出具体问题，而非泛泛而谈
- 有建设性，而非纯批评

示例：
```markdown
### 批判性思考

- 26% 成功率是否足够？距离生产可用还差什么？
- "通用性"是否牺牲"最优性"？这是正确权衡吗？
```

## 实践任务难度怎么划分？

| 难度 | 任务类型 | 时间估计 |
|------|----------|----------|
| Level 1 | 运行 demo/官方示例 | 30-60 分钟 |
| Level 2 | 修改参数/定制配置 | 1-3 小时 |
| Level 3 | 新功能/扩展实验 | 3-10 小时 |

---

## 笔记格式错误怎么修复？

常见错误：

| 错误 | 修复 |
|------|------|
| wikilink 用了绝对路径 | 改为相对路径 `[[论文]]` |
| PDF 链接不正确 | 检查 `../pdfs/` 目录 |
| frontmatter 缺字段 | 补充 `title`, `date`, `version` |
| 架构图渲染混乱 | 确保使用 ASCII 字符 |

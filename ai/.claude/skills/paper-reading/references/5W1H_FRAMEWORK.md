# 5W1H 深度思考框架

对应传统记叙文六要素，用于系统性拆解论文。

---

## What（何事）—— 论文讲的是什么？

### 核心问题

| 问题 | 输出形式 |
|------|----------|
| 核心定义是什么？ | 一句话学术定义 |
| 技术架构是什么？ | ASCII/Mermaid 架构图 |
| 核心组件是什么？ | 组件表格 |
| 关键概念是什么？ | 概念速查表 |

### 输出模板

```markdown
## What - 核心定义

{学术级定义，准确但不晦涩}

### 技术架构

```
{ASCII 架构图，展示层次和交互}
```

### 核心组件

| 组件 | 功能 | 设计理由 |
|------|------|----------|
| **{组件1}** | {...} | {...} |
```

---

## Who（何人）—— 谁做的？谁在用？

### 核心问题

| 问题 | 输出形式 |
|------|----------|
| 作者是谁？背景如何？ | 作者表格 |
| 用户是谁？为什么需要？ | 用户列表 |
| 竞品是谁？对比如何？ | 生态定位表 |
| 生态位在哪？ | 一句话定位 |

### 输出模板

```markdown
## Who - 作者与生态

| 维度 | 信息 |
|------|------|
| **作者** | {机构/背景} |
| **用户** | {目标用户} |
| **生态位** | {与竞品关系} |
```

---

## When（何时）—— 时间脉络？

### 核心问题

| 问题 | 输出形式 |
|------|----------|
| 发布时间？会议/期刊？ | 时间线图 |
| 技术演进的节点？ | 演进表格 |
| 前身工作是什么？ | 关联论文表 |
| 后续影响是什么？ | 影响说明 |

### 输出模板

```markdown
## When - 技术演进

```
2024.03 ──── {前身工作}
    │
2024.07 ──── {本论文发布}
    │
2025.05 ──── {后续发展}
```

### 技术演进脉络

| 阶段 | 代表工作 | 核心突破 |
|------|----------|----------|
| 1. {阶段名} | {工作} | {突破} |
```

---

## Where（何地）—— 在哪里发生？

### 核心问题

| 问题 | 输出形式 |
|------|----------|
| 发表在哪里？ | 平台表格 |
| 应用在哪里？ | 场景表格 |
| 代码在哪里？ | GitHub 链接 |
| 数据在哪里？ | 数据集列表 |

### 输出模板

```markdown
## Where - 平台与场景

| 类型 | 内容 |
|------|------|
| **发表** | {会议/journal} |
| **代码** | {GitHub链接} |
| **应用** | {领域场景} |
```

---

## Why（何因）—— 为什么做这个？

### 核心问题

| 问题 | 输出形式 |
|------|----------|
| 解决什么问题？ | 问题描述 |
| 现有什么缺口？ | 缺口表格 |
| 为什么这个方案？ | 设计理由 |
| 有什么学术/工业价值？ | 价值列表 |

### 输出模板

```markdown
## Why - 研究缺口

| 缺口类型 | 现有问题 | 本文方案 |
|----------|----------|----------|
| **{类型1}** | {...} | {...} |

**核心贡献**: {一句话总结}
```

---

## How（何果）—— 结果如何？怎么做到的？

### 核心问题

| 问题 | 输出形式 |
|------|----------|
| 技术细节是什么？ | 代码片段 |
| 如何验证有效性？ | 评估协议表 |
| 性能结果如何？ | 结果对比表 |
| 成本开销如何？ | 成本说明 |

### 输出模板

```markdown
## How - 技术实现

### 关键技术

#### 1. {技术名称}

```python
{代码片段或伪代码}
```

**设计理念**: {解释}

### 评估结果

| 方法 | 指标 | 结果 |
|------|------|------|
```

## TODO（后续反思）

不属于 5W1H，但同样重要的实践环节。

### 理解检验

检验是否真正理解论文核心。

```markdown
### 理解检验

- [ ] **Q1**: {核心概念问题}
- [ ] **Q2**: {技术细节问题}
```

### 批判性思考

质疑论文的假设、方法、结论。

```markdown
### 批判性思考

- {假设是否合理？}
- {方法是否有局限？}
- {结论是否可靠？}
```

### 实践任务

分难度的动手任务。

```markdown
### 实践任务

| 难度 | 任务 |
|------|------|
| Level 1 | {体验级：运行 demo} |
| Level 2 | {定制级：修改参数} |
| Level 3 | {扩展级：新功能} |
```

---

## 常见陷阱

| 陷阱 | 解决方案 |
|------|----------|
| 只抄摘要 | 必须读 Methods/Results |
| 跳过评估 | 分析评估协议是否合理 |
| 忽略限制 | 找 Limitations 章节 |
| 无批判思考 | 至少提 1 个质疑 |
| 不动手实践 | Level 1 任务必须完成 |
